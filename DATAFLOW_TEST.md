# 项目雷达数据闭环 - 测试验证文档

## 功能概述

实现了项目搜索 → 标书工厂/中标预测 的一键跳转和数据传递功能。

## 修改的文件

### 1. `index.html`（项目搜索主页）
**修改位置**: 项目卡片详情区域
**新增内容**: 
- 📄 生成标书按钮 - 跳转到标书工厂，传递项目名称、招标人、预算、原文链接
- 📊 预测中标按钮 - 跳转到中标预测，传递项目名称、预算、招标单位、地区

**URL 参数格式**:
```
标书工厂：bid_factory/index.html?projectName=xxx&tenderer=xxx&budget=xxx&link=xxx
中标预测：win_prediction.html?projectName=xxx&budget=xxx&tenderUnit=xxx&region=xxx
```

### 2. `bid_factory/app.js`（标书工厂）
**新增功能**:
- `parseUrlParams()` - 解析 URL 参数
- `fetchAnnouncementContent()` - 尝试抓取公告全文（受 CORS 限制）
- 自动填充招标公告输入框
- 用户提示信息

**工作流程**:
1. 页面加载时检查 URL 参数
2. 提取 projectName, tenderer, budget, link 参数
3. 构建公告文本模板并填充到输入框
4. 提示用户补充完整内容
5. （可选）尝试抓取原文链接内容

### 3. `win_prediction.html`（中标预测）
**新增功能**:
- `parseUrlParams()` - 解析 URL 参数
- 自动填充表单字段
- 自动触发预测（如果关键信息已填写）

**工作流程**:
1. 页面加载时检查 URL 参数
2. 提取 projectName, budget, tenderUnit, region 参数
3. 自动填充对应表单字段
4. 预算金额自动提取数字部分
5. 地区自动匹配下拉选项
6. 如果项目名称和预算已填写，自动开始预测

## 已知限制

### 1. 公告全文自动抓取
- **问题**: 由于浏览器 CORS（跨域资源共享）安全限制，无法直接从 JavaScript 抓取外部网页内容
- **解决方案**: 
  - 当前实现：提示用户手动复制粘贴
  - 生产环境建议：
    - 方案 A: 使用后端代理服务器
    - 方案 B: 使用浏览器扩展（如 CORS Unblock）
    - 方案 C: 使用第三方 CORS 代理服务

### 2. 地区匹配
- **问题**: 项目雷达的地区格式可能与中标预测的下拉选项不完全匹配
- **当前实现**: 尝试模糊匹配（检查值或文本是否包含）
- **建议**: 统一地区数据格式

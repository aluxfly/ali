## 🔍 问题分析

### 根本原因

正则表达式使用了全局标志 `/g`，导致 `String.match()` 方法行为异常：

- **有 `/g` 标志**：`match()` 返回所有匹配项的数组，**不包含捕获组**
- **无 `/g` 标志**：`match()` 返回第一个匹配项及其捕获组

示例：
```javascript
// 错误：使用 /g 标志
const pattern = /项目名称\s*[：:]\s*(.+?)(?:\n|$)/gi;
text.match(pattern); 
// 返回：['项目名称：XXX 项目'] - 没有捕获组

// 正确：不使用 /g 标志
const pattern = /项目名称\s*[：:]\s*(.+?)(?:\n|$)/i;
text.match(pattern);
// 返回：['项目名称：XXX 项目', 'XXX 项目'] - 包含捕获组
```

### 影响范围

所有 8 个解析字段均受影响：
- ❌ 项目名称
- ❌ 项目编号
- ❌ 招标人
- ❌ 预算金额
- ❌ 投标截止时间
- ❌ 项目地点
- ❌ 联系人
- ❌ 联系电话

## ✅ 修复方案

### 1. 修复正则表达式

移除所有正则表达式的全局标志 `/g`：

**修复前**：
```javascript
/项目名称 [\s]*[：:][\s]*(.+?)(?:\n|$)/gi
```

**修复后**：
```javascript
/项目名称\s*[：:]\s*(.+?)(?:\n|$)/i
```

### 2. 增强正则表达式模式

- `工程名称`
- `采购项目名称`

**招标人**：

- `采购人`

**预算金额**：

**其他字段**也类似扩展。

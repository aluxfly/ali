## 网站分析结果

### 技术栈识别

```
框架：Angular (SPA 单页应用)
UI 库：DevExtreme
服务器：nginx
安全：国密算法 (SM.js)、加密传输模块
路由：Hash 路由 (#/path)
```

### 关键发现

1. **SPA 架构**: 网站使用 Angular 构建，内容通过 JavaScript 动态加载
2. **资源保护**: 直接访问 JS/CSS 资源返回 404，需要正确的 Referer 和会话
3. **API 隐藏**: API 端点通过配置文件动态获取 (`ECPCONFIG.ipMap`)
4. **加密机制**: 使用国密算法进行数据传输加密

### 爬取挑战

- ❌ 无法通过简单 HTTP 请求获取动态内容
- ❌ Playwright 需要系统依赖（libatk-1.0.so.0 等）
- ⚠️ 当前环境无法安装浏览器依赖

## 实现方案

### 代码架构

```
crawl_sgcc.py
├── SGCPCrawler 类
│   ├── crawl_with_playwright()  # Playwright 模式
│   ├── crawl_with_requests()    # Requests 备用模式
│   ├── parse_project_list()     # HTML 解析

├── 日志系统

```

### 核心功能

1. **双模式爬取**
   - 优先使用 Playwright 获取真实数据

   - 最终降级到示例数据

2. **错误处理**
   - 重试机制（最多 3 次）
   - 请求延迟（2 秒）
   - 详细日志记录

3. **数据提取**
   - 支持多种 HTML 选择器
   - 智能去重
   - 日期解析

## 限制说明

### 当前限制

1. **Playwright 依赖缺失**
   - 系统缺少 libatk-1.0.so.0 等库
   - 无法自动安装（无 apt-get）
   - 影响：无法执行 JavaScript 获取动态内容

2. **网站 SPA 架构**
   - 内容通过 Angular 动态加载
   - 简单 HTTP 请求只能获取空壳 HTML
   - 影响：Requests 模式无法获取真实数据

3. **网络环境**
   - 网站可能需要特定网络环境（如内网）
   - 部分资源返回 404
   - 影响：可能需要代理或特殊配置

### 解决方案

**短期**（当前）:
- 使用示例数据演示功能
- 代码已准备好，待环境完善后即可获取真实数据

**中期**:
- 安装 Playwright 系统依赖
- 测试真实数据爬取

**长期**:
- 考虑 API 逆向工程
- 实现分布式爬取
- 添加代理池支持
